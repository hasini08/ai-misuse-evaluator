# AI Misuse Evaluator (Prototype)

A modular Python prototype that evaluates AI use cases using a multi-dimensional responsibility framework.
The system integrates **environmental impact (emissions)**, **ethical risk**, **creative displacement**, and **purpose justification**
into a single **Justifiability Score (0–100)** with explainable component breakdown and quantitative evaluation outputs.

## Project Goal
To provide a reproducible computational framework for comparing AI deployments and identifying use cases that are
high-cost/low-benefit or ethically risky relative to their justification.

---

## Input Dataset

The pipeline expects a CSV file at:

`data/usecases.csv`

Required columns:

- `use_case_name` (string)
- `emissions_kgco2e` (float) — environmental impact, e.g. from CodeCarbon or estimates
- `purpose_category` (categorical) — one of: `essential`, `beneficial`, `low_benefit`, `harmful`
- `ethical_risk_score` (int 1–5) — 1 = lowest risk, 5 = highest risk
- `creative_displacement_score` (int 1–5) — 1 = minimal displacement, 5 = high displacement

Optional columns:
- `model_size` (int) — for metadata only

---

## Output Files

When you run the pipeline, the following files are produced:

### Main outputs
- `data/results_scored.csv`  
  Contains the computed `justifiability_score`, labels (Low/Medium/High), and component breakdown columns.

### Analysis outputs (when enabled)
- `data/analysis_summary.txt`  
  Human-readable report including: top/bottom cases, correlations, component summary, category breakdown,
  Pareto frontier summary, sensitivity robustness, and Monte Carlo stability.

- `data/sensitivity_results.csv`  
  Sensitivity experiment varying the emissions weight and measuring ranking stability.

- `data/pareto_frontier.csv`  
  Full scored results including a boolean `pareto_optimal` column indicating non-dominated solutions.

- `data/montecarlo_stability.csv`  
  Monte Carlo robustness experiment showing how frequently each use case is top-ranked under random weight sampling.

### Visual outputs
Generated by `src/visualise.py`:
- `figures/score_bar.png`
- `figures/correlation_matrix.png`
- `figures/sensitivity_weights.png` (if sensitivity results exist)

---

## How Scoring Works (High Level)

Each use case is converted into normalised “goodness” components (0–1):
- Lower emissions → higher `emissions_norm`
- Lower ethical risk → higher `ethics_norm`
- Lower creative displacement → higher `creativity_norm`
- Higher purpose weight → higher `purpose_norm`

The Justifiability Score is computed as:

`score = 100 * weighted_sum(components)`

Weights are configurable and evaluated via sensitivity and Monte Carlo experiments.
